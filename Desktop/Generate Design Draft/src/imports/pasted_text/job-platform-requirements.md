## 求职招聘平台（类似Boss直聘）客户端开发需求文档

**说明**：本需求文档围绕一款类“Boss直聘”的求职招聘平台核心场景设计。技术栈严格限定为「前端Flutter（跨平台移动端）、后端Java Spring Boot + MySQL」，重点考察开发者的跨平台UI构建能力、Java后端分层设计、API安全控制、数据关联处理及全局异常管理能力。最终交付物需支持安卓、iOS、鸿蒙三大主流操作系统。

## 一、用户认证与个人资料管理模块

### 1. 业务场景描述

开发一款面向求职者的招聘App。用户首次使用需通过手机号注册登录。登录后，需要完善个人资料（头像、姓名、期望职位、工作城市、薪资要求）和在线简历（教育经历、工作经历）。HR用户可后续实现，本期仅实现求职者端。要求认证流程安全，资料数据规范，为后续职位推荐和投递功能提供基础。

### 2. 功能要求（严格贴合技术栈）

**后端（Java Spring Boot + MySQL）**

| 功能点            | 详细要求                                                     |
| :---------------- | :----------------------------------------------------------- |
| 用户注册          | 接收手机号、验证码（模拟：后端可生成固定验证码如“123456”）、密码。校验手机号格式及唯一性，密码需加密存储（BCrypt），返回注册成功信息。 |
| 用户登录          | 接收手机号/密码，校验成功后颁发JWT令牌（包含用户ID、角色），过期时间设为7天。 |
| 获取/修改个人资料 | 根据JWT中的用户ID查询或更新用户信息（头像URL、姓名、期望职位、城市、薪资），返回标准格式数据。 |
| 简历管理（CRUD）  | 支持对教育经历（学校、专业、学历、时间段）和工作经历（公司、职位、工作描述、时间段）的新增、修改、删除和查询。一个用户对应多条教育和工作经历。 |

**前端（Flutter）**

- **注册/登录页**：使用`TextField`构建手机号、验证码/密码输入框，表单校验（非空、手机号格式）。使用`Provider`或`GetX`管理认证状态。
- **个人资料页**：展示和编辑个人信息。头像支持从相册选取并上传（需调用后端文件上传接口）。使用`Form` + `TextFormField`进行数据录入和校验。
- **简历编辑页**：动态表单列表，支持添加/删除多条教育和工作经历。使用`ListView.builder`渲染列表。
- **令牌管理**：使用`FlutterSecureStorage`安全存储JWT。在`http`请求拦截器中自动添加`Authorization: Bearer `头。
- **401处理**：检测到401状态码时，清除本地令牌和用户状态，并导航回登录页。

### 3. 考察点

**Java后端分层设计**

- 清晰的分层架构：Controller（接收请求）、Service（业务逻辑）、Repository（数据访问）、DTO（数据传输对象）。
- 使用Spring Security + JWT实现无状态认证，自定义`UserDetailsService`和`JwtAuthenticationFilter`。

**数据校验与安全**

- 使用`Spring Validation`（`@NotNull`, `@Pattern`等）对输入参数进行校验。
- 密码使用`BCryptPasswordEncoder`加密存储。
- JWT密钥配置在`application.yml`或环境变量中。

**ORM关联映射**

- 使用Spring Data JPA或MyBatis-Plus定义`User`、`Education`、`WorkExperience`实体。
- 正确配置一对多关系（`@OneToMany`），级联操作（如用户删除时是否删除关联简历）。

**Flutter状态管理与导航**

- 使用`Provider`或`GetX`管理全局用户登录态。
- `FlutterSecureStorage`用于安全存储敏感信息。
- `dio` + 拦截器实现请求封装和令牌添加。
- `go_router`或`GetX`实现认证路由守卫（未登录无法访问个人资料和简历页）。

## 二、职位浏览与搜索模块

### 1. 业务场景描述

求职者登录后，首页展示热门职位推荐列表。用户可按关键词（职位名、公司名）、工作城市、薪资范围、学历要求等条件筛选职位。点击职位可查看详情（职位描述、公司信息、要求）。要求列表分页加载流畅，搜索响应迅速。

### 2. 功能要求

**后端（Java + MySQL）**

- **职位列表接口**：支持分页（page, size）、排序（默认按发布时间倒序）。返回职位ID、标题、公司名称、城市、薪资、学历要求、公司Logo。
- **职位搜索接口**：支持多条件组合查询（关键词模糊匹配标题/公司名、城市枚举、薪资区间、学历枚举）。使用Spring Data JPA Specification或MyBatis-Plus动态条件构造器实现。
- **职位详情接口**：根据职位ID返回完整职位描述、公司简介、HR联系方式（脱敏处理）。
- **热门职位推荐**：可简单实现为按浏览量排序或随机返回固定数量职位。

**前端（Flutter）**

- **首页职位列表**：使用`ListView.separated`构建无限滚动列表，上拉加载更多。每个卡片包含职位名、公司名、薪资、城市、公司Logo（使用`CachedNetworkImage`）。
- **筛选搜索页**：使用`BottomSheet`或独立页面展示筛选项（城市下拉、薪资范围滑块、学历单选框）。搜索结果复用职位列表组件。
- **职位详情页**：展示职位完整信息，包含“立即沟通/投递简历”按钮。
- **状态管理**：使用`RefreshIndicator`实现下拉刷新，`ScrollController`实现加载更多。

### 3. 考察点

**复杂查询与分页**

- 后端掌握动态SQL构造（Specification或LambdaQueryWrapper），避免硬编码查询逻辑。
- 使用Pageable对象统一处理分页参数，返回标准分页结构（total, pages, current, records）。

**数据库索引优化**

- 为职位表的`title`、`city`、`publish_time`字段建立合理索引，提升查询效率。

**Flutter列表性能**

- 正确使用`ListView.builder`实现按需渲染，避免一次性构建大量Widget。
- 图片缓存与占位图处理，提升列表滑动流畅度。

**前后端交互规范**

- 统一的API响应结构（`code`, `message`, `data`），分页数据嵌套在`data`中。
- 前端封装统一的HTTP服务，集中处理加载状态（如在请求前后控制`isLoading`状态，展示`CircularProgressIndicator`）。

## 三、职位收藏与投递记录模块

### 1. 业务场景描述

求职者可以收藏感兴趣的职位，方便后续查看。同时可以一键投递简历到某个职位，并查看自己的投递记录（职位、公司、投递时间、状态：已查看/邀请面试/不合适）。要求数据关联清晰，防止重复投递或收藏。

### 2. 功能要求

**后端（Java + MySQL）**

| 实体       | 关联关系                                                | 要求                                                         |
| :--------- | :------------------------------------------------------ | :----------------------------------------------------------- |
| 收藏表     | 用户ID + 职位ID（联合唯一约束）                         | 新增/取消收藏接口；查询用户收藏的职位列表（分页）。          |
| 投递记录表 | 用户ID + 职位ID（允许同一用户重复投递但可记录最新时间） | 投递接口（校验职位存在、简历完整性）；查询用户投递记录列表（分页，按投递时间倒序）。 |

- **约束处理**：重复收藏需返回友好提示（409）；投递成功后更新投递时间。
- **数据完整性**：删除职位或用户时，关联的收藏和投递记录如何处理（可选择保留或级联删除）。

**前端（Flutter）**

- **职位详情页**：显示“收藏”（☆/★ 图标切换状态）和“立即投递”按钮。
- **我的收藏页**：列表展示收藏的职位，支持左滑取消收藏（`Dismissible`）。点击跳转职位详情。
- **投递记录页**：列表展示投递记录，显示状态标签（使用不同背景色区分状态）。
- **防重复操作**：点击投递或收藏时，先检查状态，若已存在则给出Toast提示，并禁用按钮短暂时间防止重复请求。

### 3. 考察点

**数据库关联与约束**

- 正确使用联合唯一索引（`UNIQUE KEY`）防止重复收藏。
- 外键关联（`@ManyToOne`）或逻辑关联，理解JPA级联类型（CascadeType）和孤儿移除（orphanRemoval）的适用场景。

**事务管理**

- 投递操作可能涉及多表（如检查用户简历是否存在、插入投递记录），需使用`@Transactional`确保数据一致性。

**Flutter交互与手势**

- `Dismissible`实现滑动删除（取消收藏），需处理方向识别和确认对话框。
- 状态提升：职位详情页的收藏图标状态需实时反映最新状态，且修改后返回收藏列表页应自动刷新。

**状态同步**

- 使用`ChangeNotifier`或`BLoC`跨页面同步状态（例如在收藏列表页取消收藏后，回到详情页时收藏图标也应变化）。

## 四、全局日志与异常处理规范

### 1. 业务场景描述

为便于线上问题排查和监控，需实现全栈统一的日志和异常处理规范。后端记录所有API请求的入参、出参、耗时、IP及异常堆栈，前端记录关键用户行为和系统异常。全局异常处理器确保所有错误返回标准JSON格式，避免敏感信息泄露。

### 2. 功能要求

**后端（Spring Boot）**

- **全局日志中间件**：
  - 使用`HandlerInterceptor`或`OncePerRequestFilter`记录每个请求的：请求URI、方法、客户端IP、请求参数（或Body摘要）、响应状态码、执行时间。
  - 使用SLF4J + Logback输出日志，按天滚动，区分`info.log`和`error.log`。
- **全局异常处理器**：
  - 使用`@RestControllerAdvice`捕获所有未处理异常。
  - 自定义业务异常（如`BusinessException`）和特定异常（`MethodArgumentNotValidException`, `HttpMessageNotReadableException`, `DataIntegrityViolationException`等）。
  - 返回统一格式：`{code, message, data}`，code遵循HTTP状态码语义（如400参数错误，401未授权，404不存在，409冲突，500系统错误）。
- **敏感信息过滤**：日志中自动脱敏手机号、密码字段（使用正则替换）。

**前端（Flutter）**

- **统一请求响应处理**：
  - 封装Dio实例，使用拦截器统一处理请求头（添加JWT）、打印请求/响应日志（调试模式）。
  - 根据后端返回的`code`字段全局处理：如`code != 200`时弹出SnackBar提示`message`；`code == 401`时清除用户状态并跳转登录页。
- **错误上报**（加分项）：实现`FlutterError.onError`和`runZonedGuarded`捕获框架级错误，并将错误栈和用户操作路径上报至后端日志收集接口。

### 3. 考察点

**Spring Boot拦截器/AOP**：掌握拦截器实现与注册，理解`preHandle`、`postHandle`、`afterCompletion`的执行顺序。或使用AOP `@Around`实现更精细的日志记录。

**全局异常处理设计**：

- 区分业务异常、系统异常、第三方异常，返回用户友好的错误信息，不暴露数据库细节或堆栈。
- 对参数校验异常（`BindException`）统一返回字段错误详情。

**Flutter错误处理**：

- 使用`try-catch`包裹异步调用，并正确处理`onError`回调。
- 全局`ErrorWidget.builder`自定义加载失败时的占位图。

**日志规范与安全**：

- 日志需包含时间戳、线程、类名、级别。
- 生产环境禁止打印敏感信息（如密码原文、完整身份证号），配置logback的过滤器实现脱敏。

## 补充说明

### 1. 技术栈约束

| 层级       | 技术选型                        | 约束说明                                                     |
| :--------- | :------------------------------ | :----------------------------------------------------------- |
| 前端       | Flutter (3.16+)                 | 使用Dart语言，一套代码打包生成Android / iOS / HarmonyOS（鸿蒙）三端应用。 |
| 后端       | Java 17 / Spring Boot 3.x       | 推荐使用Spring Boot + Spring Security + JWT + Spring Data JPA / MyBatis-Plus。 |
| 数据库     | MySQL 8.0+                      | 使用InnoDB引擎，字符集utf8mb4。                              |
| 持久层框架 | Spring Data JPA 或 MyBatis-Plus | 禁止手写JDBC或拼接SQL。                                      |
| HTTP客户端 | Dio                             | Flutter端统一使用Dio。                                       |

### 2. 跨平台打包要求

- **Android**：生成release APK或AAB，最低支持API 21（Android 5.0）。
- **iOS**：生成ipa文件，支持iOS 12.0+。需提供相关配置文件（需在Mac上打包）。
- **鸿蒙（HarmonyOS）**：使用Flutter鸿蒙适配版本（如OpenHarmony Flutter），生成.hap包。若环境受限，可提供详细的打包配置文档和未签名包作为证明。

### 3. 代码与文档规范

- **后端**：遵循阿里巴巴Java开发手册，包结构清晰（`controller`, `service`, `repository`, `dto`, `entity`, `config`, `util`），关键类和方法有注释。
- **前端**：遵循Flutter官方代码规范，组件化拆分，使用`const`优化重建，避免嵌套过深。
- **API文档**：要求提供Swagger/OpenAPI（SpringDoc）自动生成接口文档，启动后可通过`/swagger-ui.html`访问。
- **启动说明**：提供`README.md`，包含后端数据库初始化脚本、Redis（可选）配置、JWT密钥配置，前端环境变量（API_BASE_URL）配置方法。

### 4. 加分项

- **容器化部署**：提供Dockerfile和docker-compose.yml一键启动后端+MySQL。
- **单元测试**：后端编写至少5个核心Service的JUnit测试用例；前端编写Widget测试或单元测试。
- **状态管理进阶**：使用Riverpod或BLoC替代简单的Provider，体现复杂状态流转管理。
- **离线缓存**：职位列表和收藏列表使用Hive或SQLite做本地缓存，优化弱网体验。

### 5. 提交物

- 前端Flutter完整项目源码。
- 后端Java Spring Boot完整项目源码。
- 数据库建表及初始化数据的SQL脚本。
- 项目README及操作演示视频（或截图序列）。
- 各平台的打包产物（APK、通用ipa包、鸿蒙.hap包）或明确的打包产出证明。
